## 📋 About This System

A fully offline-capable Data Management System for the **West New Britain Provincial Administration, Community Development Division**, developed by **Naitechs Prod**.

### ✅ Features
- 15 data modules (Civil Registry, Marriage, Adoption, Death, Disability, Women's Group, Youth Group, Child Protection, Labour Mobility, Quarterly Tracker & more)
- Role-based access (Boss / Officer)
- Local browser storage — works 100% offline
- Excel export (XLSX)
- Print to PDF for all records
- User management
- Audit/activity log
- Broadcast announcements

## 🌐 STEP-BY-STEP: Host FREE on GitHub Pages

### Prerequisites
- A free GitHub account → [github.com](https://github.com)
- The `index.html` file from this repository

---

### STEP 1 — Create a GitHub Account (if you don't have one)

1. Go to **[github.com](https://github.com)**
2. Click **Sign up**
3. Enter your email, create a password, choose a username
4. Verify your email address
5. Choose the **Free** plan

---

### STEP 2 — Create a New Repository

1. After logging in, click the **"+"** button (top-right) → **New repository**
2. Fill in the form:
   - **Repository name:** `wnb-cdd-dms` (or any name you like)
   - **Description:** `WNB CDD Data Management System`
   - **Visibility:** ✅ Public *(must be Public for free GitHub Pages)*
   - **Initialize:** ✅ Check "Add a README file"
3. Click **Create repository**

---

### STEP 3 — Upload the Files

**Option A — Using the GitHub Website (Easiest):**

1. In your new repository, click **Add file** → **Upload files**
2. Drag and drop `index.html` into the upload area
3. Scroll down, write a commit message: `Add WNB CDD DMS`
4. Click **Commit changes**

**Option B — Using Git (Command Line):**

```bash
# Clone your repo
git clone https://github.com/YOUR-USERNAME/wnb-cdd-dms.git
cd wnb-cdd-dms

# Copy index.html into this folder, then:
git add index.html
git commit -m "Add WNB CDD DMS v2.0"
git push origin main
```

---

### STEP 4 — Enable GitHub Pages

1. In your repository, click **Settings** (top menu bar)
2. Scroll down the left sidebar → click **Pages**
3. Under **Source**, select:
   - Branch: **main**
   - Folder: **/ (root)**
4. Click **Save**
5. Wait 1–2 minutes
6. GitHub will show: *"Your site is published at https://YOUR-USERNAME.github.io/wnb-cdd-dms/"*

---

### STEP 5 — Access Your Live Website

Your DMS is now live at:
```
https://YOUR-USERNAME.github.io/wnb-cdd-dms/
```

**Share this URL** with all CDD officers — it works on any device with a browser!

## 💾 Data Storage

This GitHub Pages edition stores all data in the **browser's localStorage**.

| Feature | Details |
|---------|---------|
| Storage location | Browser (device-local) |
| Max storage | ~5-10 MB per browser |
| Data persistence | Stays until browser data is cleared |
| Multi-device sync | ❌ Each device has its own data |
| Offline access | ✅ Works without internet |

### 📤 Backing Up Data
Use the **📥 Excel** export button in each module to download your data as `.xlsx` files regularly.

## 🔄 Updating the System

To update when a new version is available:

1. Download the new `index.html`
2. Go to your GitHub repository
3. Click on `index.html` → click the ✏️ pencil (edit) icon → or re-upload the file
4. Commit the change
5. GitHub Pages updates automatically within 1–2 minutes

## ❓ Troubleshooting

| Problem | Solution |
|---------|---------|
| Page not loading | Wait 5 minutes after enabling Pages, then refresh |
| Data disappeared | Data is per-browser — don't clear browser data without exporting first |
| Can't log in | Use `admin` / `admin123` (default Boss account) |
| Site shows 404 | Make sure `index.html` is in the **root** of your repository (not in a subfolder) |
| Changes not appearing | Hard refresh: `Ctrl+Shift+R` (Windows) or `Cmd+Shift+R` (Mac) |
